# Autogenerate Columns

Automatically generate table columns from your datasource without manually declaring each column.

## Basic Usage

Instead of manually declaring columns:

```html
<ui-table-view [datasource]="adapter">
  <ui-text-column key="firstName" headerText="First Name"></ui-text-column>
  <ui-text-column key="lastName" headerText="Last Name"></ui-text-column>
  <ui-text-column key="email" headerText="Email"></ui-text-column>
</ui-table-view>
```

Use the directive to auto-generate them:

```html
<ui-table-view [datasource]="adapter" uiAutogenerateColumns></ui-table-view>
```

## How it Works

The `uiAutogenerateColumns` directive:

1. **Introspects** the first row in your datasource
2. **Extracts** all property keys
3. **Creates** a `UITextColumn` for each property
4. **Sorts** columns alphabetically for consistent ordering
5. **Humanizes** header text by default (e.g. `firstName` → `First Name`)

## Options

Customize behavior with configuration:

```html
<ui-table-view
  [datasource]="adapter"
  [uiAutogenerateColumns]="config()"
></ui-table-view>
```

### humanizeHeaders (boolean, default: true)

Controls whether property names are converted to human-readable labels.

```ts
// Default behavior - uses humanized headers
[uiAutogenerateColumns] = "{ humanizeHeaders: true }";
// firstName → First Name
// user_email → User Email
```

Disable to use property names as-is:

```ts
[uiAutogenerateColumns] = "{ humanizeHeaders: false }";
// firstName → firstName
// user_email → user_email
```

### headerMap (Record<string, string>)

Override header text for specific columns:

```ts
[uiAutogenerateColumns]="{
  headerMap: {
    firstName: 'Given Name',
    lastName: 'Family Name',
    userEmail: 'Email Address'
  }
}"
```

### excludeKeys (string[])

Hide specific properties from the generated columns:

```ts
[uiAutogenerateColumns]="{
  excludeKeys: ['id', '_internal', 'metadata']
}"
```

## Example with all options

```typescript
export class MyComponent {
  readonly datasource = signal(/* ... */);

  readonly autoGenerateConfig = signal({
    excludeKeys: ["id", "createdAt"],
    headerMap: {
      firstName: "First Name",
      lastName: "Last Name",
    },
    humanizeHeaders: true,
  });
}
```

```html
<ui-table-view
  [datasource]="datasource()"
  [uiAutogenerateColumns]="autoGenerateConfig()"
></ui-table-view>
```

## Text Rendering

Autogenerated columns use `UITextColumn` to display values as plain text. For more complex rendering (badges, numbers, custom templates), manually declare those columns instead.

## Mixing Manual and Auto-generated Columns

You can mix manual and auto-generated columns. Manual columns take precedence:

```html
<ui-table-view [datasource]="adapter" [uiAutogenerateColumns]="config()">
  <!-- This column is rendered manually -->
  <ui-badge-column key="status" headerText="Status"></ui-badge-column>
</ui-table-view>
```

## When to Use

- ✅ **Quick prototyping** — Display data without setup boilerplate
- ✅ **Dynamic data structures** — When you don't know the schema ahead of time
- ✅ **Admin tools** — Generic data viewing interfaces
- ❌ **Custom rendering** — Use manual columns for specialized cell templates
- ❌ **Complex layouts** — Mixing multiple column types works, but explicit is clearer
